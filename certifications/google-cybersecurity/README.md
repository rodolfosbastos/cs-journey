## Key Concepts by Course

### Course 1 — Foundations of Cybersecurity

**Module 1:** Introduction to cybersecurity concepts.

**Module 2:** History of cybersecurity (Morris Worm 1988, Love Bug 2000, major breaches); eight CISSP security domains (Security & Risk Management, Asset Security, Security Architecture & Engineering, Communication & Network Security, IAM, Security Assessment & Testing, Security Operations, Software Development Security).

**Module 3: Frameworks, Controls & Ethics**

*CIA Triad* — the three pillars every security decision maps to:
- **Confidentiality** — only authorised people can access data (encryption, access controls)
- **Integrity** — data is accurate and untampered (hashing, audit logs)
- **Availability** — systems are accessible when needed (redundancy, backups)

*NIST Cybersecurity Framework (CSF)* — 5 core functions (IPDRR):
1. **Identify** — know your assets and risks
2. **Protect** — implement safeguards
3. **Detect** — monitor for anomalies
4. **Respond** — contain and analyse incidents
5. **Recover** — restore systems; learn from the incident

*NIST RMF* — 7-step risk management process for federal systems: Prepare → Categorize → Select → Implement → Assess → Authorize → Monitor.

*Other frameworks:* ISO/IEC 27001 (international ISMS standard), SOC 2, FedRAMP.

*Security Controls* — safeguards to reduce risk, categorised two ways:
- By type: **Technical** (firewalls, MFA, encryption) / **Operational** (training, incident plans) / **Managerial** (policies, audits, risk assessments)
- By function: **Preventive** / **Detective** / **Corrective** / **Deterrent**

*Ethics* — key principle: having **access** does not mean having **authorization**. Always act within scope, protect privacy, follow laws, and report vulnerabilities through proper channels.
